BRANCHING PROCEDURE

This project contains a main branch, which holds the complete, updated version of the game, and branches for each team member. These working branches are all named after the member working on them, however because of early
Git problems the orginal branches needed to be scrapped and replaced with better branches, so all current working branches are named as Better[NAME]Branch. Whenever someone on the team pushes to the repository, they update
their respective working branches, then merge their branches with the main branch when code needs to be shared between different people.
